# Process Rules

Use the best matching skill, MCP, or local context source for the task.

## Selection order
1. Use a relevant skill when one exists for the task.
2. Use the most appropriate MCP or connector when the source is authoritative.
3. Use local repository context when the answer is repo-specific.
4. Use Context7 first for current library, SDK, API, CLI, and cloud documentation.

## Rules
- Do not guess when an authoritative documentation source is available.
- Prefer primary or official documentation over memory.
- If the task touches external APIs or libraries, verify the latest docs before acting.
- If a skill applies, use it before you act.
